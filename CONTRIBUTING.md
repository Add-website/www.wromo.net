# Wromo Contributor Guide

Welcome to the first web infrastructure where your code gets you actions!

### 📊 How do we measure contribution?
We use the **SourceCred** algorithm to generate an impact score. The weight of the score is:
* **Core Code (Logic):** 50% of the weight (Cloud Functions, WromoLoader).
* **Bug Fixing:** 20% of the weight.
* **UI/UX & Themes:** 20% of the weight (New store designs).
* **Documentation & Community:** 10% of the weight.

### 🛠 Workflow
1. **Fork & Branch:** Work on a separate branch.
2. **Atomic Commits:** Each commit must solve a single problem.
3. **Review:** Any PR requires at least one review from a "Core" member before going live.
4. **Minting Cred:** Once the PR is accepted, SourceCred will automatically allocate you "participation points".

### Revenue Structure:
* **Operational Costs** (30%): Google Cloud Servers, Github, domain, APIs.

* **Reserve Fund** (10%): For emergencies or marketing.

* **Contributor Distribution** (60%): This money is divided proportionally to the Credit score accumulated by each programmer.

### How do you vote?
* **Major decisions**  are made by weighted vote: 1 Credit Point = 1 Vote.
